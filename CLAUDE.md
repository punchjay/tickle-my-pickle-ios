# CLAUDE.md

Guidance for working in this repo. See `README.md` for the full project overview.

## What this is

Native SwiftUI pickleball court finder (iOS 17+, Swift 6 strict concurrency). Ground-up
rewrite of the older React/Expo app — shares no code with it.

## Map vs. search — a deliberate split (don't "fix" this)

- **The map is Apple MapKit**, on purpose. No API key, no Google Maps SDK. The map you
  see is Apple Maps; this is correct, not a bug. Seeing Apple Maps instead of Google Maps
  is the intended design.
- **Google is used only for search data:** Geocoding (text → coordinates) and Places API
  (New) text search (find courts). See `TickleMyPickle/Logic/GooglePlacesClient.swift`.

## Build & run (no Xcode GUI needed)

Project file is generated by XcodeGen from `project.yml` (the `.xcodeproj` is gitignored).
After changing `project.yml`, run `xcodegen generate`. Changing only `Secrets.xcconfig`
does NOT require regenerating — just rebuild.

```bash
xcodebuild -project TickleMyPickle.xcodeproj -scheme TickleMyPickle -configuration Debug \
  -destination 'platform=iOS Simulator,name=iPhone 17,OS=latest' -derivedDataPath build build
xcrun simctl boot "iPhone 17" 2>/dev/null || true
xcrun simctl bootstatus "iPhone 17" -b
xcrun simctl install booted build/Build/Products/Debug-iphonesimulator/TickleMyPickle.app
xcrun simctl launch booted com.punchjay.ticklemypickle
```

For a fast edit loop, `./refresh.sh` wraps build + install + relaunch (fails fast
on build errors so it never relaunches a stale app). Override the target sim with
`SIM="iPhone 17 Pro" ./refresh.sh`.

Tests: `xcodebuild test -project TickleMyPickle.xcodeproj -scheme TickleMyPickle -destination '...'`

## Google Maps API key (the setup that's easy to get wrong)

The key goes in `Secrets.xcconfig` (gitignored) as `GOOGLE_MAPS_REST_API_KEY`, substituted
into Info.plist at build time via `project.yml`. `Config.swift` reads it back.

The key **must** be:
- **iOS-app restricted** to bundle id `com.punchjay.ticklemypickle` — NOT HTTP-referrer
  restricted. `GooglePlacesClient` sends `X-Ios-Bundle-Identifier` on every request and no
  referrer, so a referrer-restricted (web) key returns `REQUEST_DENIED`.
- **API-restricted to Geocoding API + Places API (New)** — and both must be enabled on the
  project. If the key is accidentally restricted to some other API only (e.g. "API Keys
  API"), every search fails with `REQUEST_DENIED` / "not authorized to use this service".

The correct key lives in the Google Cloud project **"My First Project"**
(`project-6f338a57-964a-40ff-b35`) as **"API key 2"**. That project also holds the **separate**
web/React app key ("Maps Platform Web Key", referrer-restricted) — leave it alone. Ignore
the empty GCP project `718145742726`; the iOS key is not there.

Quick diagnosis when search fails — hit the endpoints directly with the app's header:

```bash
curl -s -H "X-Ios-Bundle-Identifier: com.punchjay.ticklemypickle" \
  "https://maps.googleapis.com/maps/api/geocode/json?address=Denver&components=country:US&key=$KEY"
```

Debug builds surface Google's real error on-screen; release shows a friendly message
(see `PickleballMapViewModel.searchNearby`).
